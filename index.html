Criar a tabela manualmente no Supabase
A criação automática da tabela depende de uma função (rpc/exec_sql) que não existe por padrão em projetos Supabase — por isso o sistema não consegue criá-la sozinho. Resolva em 3 passos:
  1. Acesse seu projeto em supabase.com → menu SQL Editor.
  2. Clique em New query, cole o script abaixo e clique em Run.
  3. Volte aqui e clique em Verificar tabela.
CREATE TABLE IF NOT EXISTS public.carregamentos_bi (
  id BIGSERIAL PRIMARY KEY,
  ref TEXT NOT NULL,
  transportadora TEXT,
  destino TEXT,
  data_carga DATE,
  volumes TEXT,
  peso TEXT,
  qtd_veiculos INT DEFAULT 1,
  prioridade TEXT DEFAULT 'normal',
  obs TEXT,
  itens_pedido TEXT,
  processado BOOLEAN DEFAULT FALSE,
  status TEXT DEFAULT 'aguardando',
  concluido_em TIMESTAMPTZ,
  criado_em TIMESTAMPTZ DEFAULT now()
);

ALTER TABLE public.carregamentos_bi ENABLE ROW LEVEL SECURITY;

DROP POLICY IF EXISTS "allow_all_carregamentos_bi" ON public.carregamentos_bi;
CREATE POLICY "allow_all_carregamentos_bi" ON public.carregamentos_bi FOR ALL USING (true) WITH CHECK (true);
Observação: a política de segurança acima (FOR ALL USING (true)) libera leitura/escrita pública na tabela. É adequada apenas porque o app usa a chave de serviço; para um uso mais seguro, restrinja a policy conforme a necessidade do projeto.
Se a tabela carregamentos_bi já existia antes do recurso "Solicitar Novo Pedido" do Portal do Cliente, o sistema tenta aplicar esta migração sozinho toda vez que você clica em Verificar tabela. Se o seu projeto Supabase não tiver a função rpc/exec_sql habilitada, rode manualmente no SQL Editor: ALTER TABLE public.carregamentos_bi ADD COLUMN IF NOT EXISTS itens_pedido TEXT; — sem essa coluna, os pedidos do cliente continuam funcionando normalmente, apenas com os itens gravados em texto dentro da observação.
O mesmo vale para ALTER TABLE public.carregamentos_bi ADD COLUMN IF NOT EXISTS concluido_em TIMESTAMPTZ; — usada para manter o carregamento concluído visível no Portal do Cliente por 7 dias e excluí-lo automaticamente depois.
